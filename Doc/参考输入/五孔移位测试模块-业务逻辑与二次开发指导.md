# 五孔移位测试模块 — 业务逻辑总结与二次开发指导

## 一、模块概述

五孔移位测试（Five-Hole Traversal Test）是本系统的核心功能模块，用于**风洞流场测量实验**。其核心流程是：使用已校准的五孔探针，按预设轨迹自动遍历多个空间点位，在每个点位采集五孔压力数据，通过PRB校准系数进行实时插值计算，获取流场参数（攻角α、侧滑角β、马赫数Ma、速度V、总压P0、静压Ps）。

**前置条件**：必须先完成探针校准（在 `FiveHoleMain.vue` 校准模块中完成），生成PRB校准文件。

## 四、核心业务逻辑详解

### 4.1 测试全生命周期 (`TraversalTestService`)

```
idle → [start] → running → [pause] → paused → [resume] → running
                                    → [stop]  → stopped
                  → [完成] → completed
                  → [错误] → error
```

**启动流程** (`start()`):
1. 获取工作流锁 (`workflow:traversal`)
2. 检查前置条件（运动控制器连接、PRB文件加载、采集设备就绪）
3. 根据布点配置生成测试点位列表
4. 初始化CSV写入器
5. 进入测试主循环

**单点测试流程** (`runSinglePoint()`):
1. **移动阶段** (`moving`)：向运动控制器发送移动指令，等待运动完成（超时30s）
2. **稳定阶段** (`stabilizing`)：等待驻留时间（`dwellTimeMs`），确保气流稳定
3. **采集阶段** (`acquiring`)：批量采集N个样本（`samplesPerPoint`），等待最新数据快照，计算平均值
4. **插值计算**：将平均压力输入插值器，计算α、β、Ma、V、P0、Ps
5. **数据持久化**：将数据点追加写入CSV文件

**错误处理**：5种专用错误类型
- `TraversalMotionError` — 运动控制失败
- `TraversalSamplingError` — 数据采集失败
- `TraversalPersistenceError` — CSV写入失败
- `TraversalInterpolationError` — 插值计算失败
- `TraversalCancelledError` — 用户取消

### 4.2 布点模式与点位生成

支持4种布点模式（`TraversalPattern`）：

| 模式 | 参数 | 点位生成逻辑 |
|------|------|-------------|
| `line` | startX/Y, endX/Y, xStepSegments, yStepSegments | 沿直线方向，按X/Y分段步长生成点位 |
| `rectangle` | xMin/Max, yMin/Max, xStepSegments, yStepSegments | 矩形网格，X/Y方向各自分段步长 |
| `sector` | centerX/Y, radiusMin/Max, angleStart/End, radial/angularSegments | 扇形区域，径向+角度分段步长，极坐标→直角坐标 |
| `custom` | points[] | 用户手动指定任意点位 |

**分段步长** (`StepSegment`)：每个维度可配置多个区段，每段有独立的start/end/step，支持变步长布点。

### 4.3 PRB插值算法 (`PrbInterpolator`)

这是模块的**核心算法**，实现了五孔探针的标准校准插值方法：

**PRB文件格式**：
- 首行：`13 13`（网格维度，α×β各13个点，-30°~+30°步长5°）
- 数据行：`ka kb cpt cps alpha beta`（6列，共169行）

**插值流程**：
1. **计算压力系数**：
   - `Ka = (P4 - P5) / ΔP`（攻角系数）
   - `Kb = (P3 - P1) / ΔP`（侧滑角系数）
   - `ΔP = P2 - (P1+P3+P4+P5)/4`（参考压力差，带最小值钳位）

2. **区域判定与角度解析**（9区域模型）：
   - 区域1/3/5/7：四角区域，直接映射到边界角度
   - 区域2/4/6/8：边界区域，沿边界线插值
   - 区域9：内部区域，在四边形单元内用距离比插值

3. **输出插值**：在解析出的(α,β)位置，对CPT和CPS进行双线性插值

4. **物理量计算**：
   - 总压：`P0 = P2 - CPT × ΔP`
   - 静压：`Ps = Pavg - CPS × ΔP`
   - 速度：`V = √(2|P0-Ps|·R·T / (Patm+Ps))`
   - 马赫数：`Ma = √(5·|((P0+Patm)/(Ps+Patm))^(2/7) - 1|)`

5. **有效性校验**：检查结果有限性、P0>Ps、角度在PRB范围内、温度/压力/密度物理合理性

### 4.4 多PRB插值 (`MultiPrbInterpolator`)

支持加载多个不同马赫数的PRB文件，实现跨马赫数插值：

**流程**：
1. 用中间马赫数的PRB文件计算初始Ma值
2. 根据Ma值选择插值策略：
   - **最近邻模式** (`nearest`)：选择Ma最接近的PRB文件
   - **线性插值模式** (`linear`)：在两个相邻Ma的PRB文件间线性插值各参数

### 4.5 通道映射与数据采集

**探针通道预设**（7个必需通道）：

| 通道名 | 角色 | 默认通道索引 |
|--------|------|-------------|
| P1 | `fiveHole.p1` | 0 |
| P2 | `fiveHole.p2` | 1 |
| P3 | `fiveHole.p3` | 2 |
| P4 | `fiveHole.p4` | 3 |
| P5 | `fiveHole.p5` | 4 |
| Patm | `fiveHole.pAtm` | 16 |
| Tatm | `fiveHole.tAtm` | 17 |

**数据采集策略**：
- 批量等待最新快照（超时2s，轮询10ms）
- 确保数据新鲜度（时间戳递增）
- 多次采样取平均值

**运动轴映射**：X/Y轴可映射到α/β，支持缩放(`scale`)和偏移(`offset`)：`targetPosition = sourceValue × scale + offset`

## 五、关键数据流图

```
用户点击"开始测试"
  → TraversalMain.startTest()
    → traversalStore.checkPreconditions()  [检查前置条件]
    → traversalStore.startTest(config)
      → traversalApi.start(config)  [IPC调用]
        → TraversalTestService.start(config)
          → checkPreconditions()  [主进程侧检查]
          → generatePoints(layout)  [生成点位列表]
          → TraversalCsvWriter.initialize()  [创建CSV文件]
          → runTestLoop()  [进入主循环]
            → for each point:
              → runSinglePoint()
                → moveToTraversalPoint()  [运动控制]
                → sleep(dwellTimeMs)  [等待稳定]
                → collectAveragedPressures()  [数据采集]
                → interpolator.calculate()  [插值计算]
                → csvWriter.appendPoint()  [写入CSV]
              → emit('progress', event)  [发射进度事件]
          → emit('complete', event)  [发射完成事件]
```

**实时数据流**：

```
DAQ设备 → onSnapshot → TraversalMain.vue → buildRealtimePressuresFromSnapshots()
  → liveInterpolationInput → syncRealtimeInterpolation() → IPC → PrbInterpolator.calculate()
  → realtimeResult → UI显示
```

- `ACQUISITION_BATCH_TIMEOUT_MS` (2000ms) — 批量采集超时
- `ACQUISITION_BATCH_POLL_INTERVAL_MS` (10ms) — 轮询间隔
