## Design Context

### Users
航空航天/流体力学领域的研究人员和工程师。使用场景包括：实验室现场操作（站在设备旁一边操作硬件一边看界面）和办公室远程监控（查看采集数据和测试进度）。他们需要的是精准的测量数据、可靠的校准结果和流畅的测试工作流。

### Brand Personality
**精确 / 高效 / 探索**。三个关键词：Precise, Efficient, Exploratory。界面应该在科幻感的外表下传递专业信任——让用户相信数据是准确的，系统是稳定的，操作是高效的。不是炫技式的科技感，而是"我用的工具很先进"的自信感。

### Aesthetic Direction
**深空霓虹 (Deep Space Neon)**。暗黑模式专属，近黑蓝底色 (#0a0a1a) 搭配霓虹紫 (#b829ff) 和霓虹青 (#00f5ff) 双主色调。玻璃拟态（Glassmorphism）为核心视觉语言：半透明背景 + backdrop-filter 模糊 + 柔和白色边框。交互元素使用发光 box-shadow 提供状态反馈。网格背景增添科幻氛围。

- **主色调**: Purple (#b829ff) + Cyan (#00f5ff)
- **功能色**: Success 绿 (#00ff88), Warning 橙 (#ffaa00), Danger 红 (#ff3366), Info 蓝 (#00aaff)
- **轴色约定**: X=紫, Y=青, Z=绿, U=橙 (不可更改)
- **字体**: Microsoft YaHei 基础，等宽数字用于数值显示，全部 UI 文字使用中文
- **组件库**: Element Plus + 深色主题覆盖，自定义 GlassCard / StatusIndicator / ValueDisplay / ChartPanel

**参考**: 科幻HUD界面、数据监控仪表盘、Blade Runner风格控制台
**反参考**: 避免花哨装饰过度、避免90年代工业控制冰冷风格、避免在一个屏幕堆砌太多数据

### Design Principles

1. **数据优先，装饰克制** — 界面中最重要的元素永远是数据和数值。霓虹发光、玻璃效果服务于信息层级，不掩盖内容。每个视觉元素都必须有功能性目的。

2. **状态即时可见** — 设备连接、采集进度、测试状态必须一目了然。StatusIndicator 的颜色和脉冲动画让用户在实验室嘈杂环境中也能快速判断系统状态。不依赖用户主动查找信息。

3. **操作流畅无阻塞** — 长时间运行的校准/测试任务通过进度条和实时数据更新提供反馈。按钮禁用状态明确，操作结果通过颜色和动画即时传达。避免让用户等待而无感知。

4. **一致的视觉语法** — 紫色=主操作，青色=数据/信息，绿色=成功/完成，橙色=警告/运动中，红色=错误/危险。轴色固定映射。GlassCard 是唯一的内容容器。相同的交互模式在整个应用中保持一致。

5. **实验室友好** — 字体足够大，关键数值使用 tabular-nums 防跳动，色彩对比度满足深色模式可读性要求。支持用户在实验室内外不同光线环境下使用。
