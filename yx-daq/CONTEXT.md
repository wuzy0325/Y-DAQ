# Y-DAQ

数据采集与运动控制桌面应用。通过多通道采集设备获取实时测量数据，结合运动控制器进行移位布点插值测试。

## 语言

### 核心概念

**采集设备 (DAQ Device)**:
通过 TCP 协议连接的硬件数据采集单元，提供多通道实时数据流。
_避免_: 采集卡、数据盒

**运动控制器 (Motion Controller)**:
通过 TCP 协议控制多轴位移的平台控制器（B140 型号）。
_避免_: 运动卡、驱动

**通道 (Channel)**:
采集设备上的单一数据源，以物理单位（Pa、V、rps 等）产出实时数值。
_避免_: 测点、端口

**数据快照 (Data Snapshot)**:
所有采集通道在某一时刻的完整读数集合（~20Hz 推送至前端）。
_避免_: 数据包、采样

### 测试类型

**五孔探针校准 (Five-Hole Probe Calibration)**:
使用移测机构（编码器补偿 + 球罐闸门）在风洞/流场中校准五孔探头的系数。
_避免_: 五孔测试、校准

**三孔移位插值测试 (Three-Hole Traversal Interpolation Test)**:
受控于运动控制器，沿矩形/直线/自定义轨迹移动三孔探针，采集各位置数据并生成插值结果。
_避免_: 三孔测试、遍历测试

### 硬件与数据

**DAQ-16**:
16 通道数据采集设备，支持 DAQ8（8 通道）和 DAQ16（16 通道）两种模式。
_避免_: XY-DAQ

**Simulated Device**:
用于开发/测试的软件模拟采集设备，在无物理硬件时产生仿真数据。
_避免_: 模拟器、假设备

**通道类型**:
- **Kalpha**: 通道 1-4（差分压力，Pa）
- **Kbeta**: 通道 5-8（差分压力，Pa）
- **CPS**: 通道 9-12（压力，Pa）
- **CPT**: 通道 13-16（压力，Pa）

**插值算法 (Interpolator)**:
基于实测离散点数据，通过 2D 插值计算未测量位置的期望值。
_避免_: 预测、推算

### 状态与周期

**服务生命周期**:
所有采集/测试服务统一实现 Start → Pause → Resume → Stop 状态机。
_避免_: 启停流程

**布点模式 (Traversal Pattern)**:
探针在测试区域的移动方式 — 矩形布点、直线布点、自定义布点。
_避免_: 扫描方式、路径

## 关系

- 一个**采集设备**包含 8 或 16 个**通道**
- 一个**运动控制器**控制 1-4 个轴（X 紫、Y 青、Z 绿、U 橙）
- **三孔移位插值测试**使用**运动控制器**移动**探针**，通过**采集设备**在每个位置读数，最终经**插值算法**输出结果
- **五孔探针校准**涉及**编码器补偿**和**球罐闸门**两个子组件

## 示例对话

> **Dev:** "用户想在三孔测试中改用五孔探针，需要新增探头类型配置吗？"
> **Domain:** "五孔探针校准是一个**独立领域**，和三孔测试使用不同的公式和流程。当前五孔相关功能已实现但被隐藏，不应和三孔测试耦合。"

## 已标记的歧义

- "五孔测试" —— 歧义：实际指"五孔探针校准"，和三孔移位插值测试是完全不同的业务领域。已用全称"五孔探针校准"区分。
- "模拟器" —— 歧义：SimulatedDevice 是纯软件模拟，不是对真实设备的抽象包装。
